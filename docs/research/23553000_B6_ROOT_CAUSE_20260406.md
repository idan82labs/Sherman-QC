# 23553000 `CAD_B6` Root Cause — 2026-04-06

## Problem

On `/Users/idant/Downloads/23553000_B_FULL_SCAN 1.ply`, `CAD_B6` was drifting badly:
- shipped result before fix: `111.42°`, `FAIL`
- expected family value from CAD / drawing: `120°`

## What was ruled out

Not the scan:
- aligned local support existed around the bend

Not merge selection:
- the final report was already using the local match, not the bad global one

Not `probe_region`:
- it produced about `113.55°` with low confidence

## Actual cause

The local `surface_classification` estimator was using too broad a locality window for this short obtuse wing bend.

Evidence sweep:
- `/Users/idant/82Labs/Sherman QC/Full Project-qc-clean/output/23553000_b6_surface_sweep_20260406.json`

Key observations:
- broad local window kept the estimate around `111°`
- narrower windows moved the estimate onto the expected family:
  - `119.99°` at `search_radius=30.0`
  - `120.17°` at `search_radius=26.0`

## Fix

Code:
- `/Users/idant/82Labs/Sherman QC/Full Project-qc-clean/backend/bend_detector.py`

Behavior:
- for selected short obtuse bends only
- if the first successful `surface_classification` result is still more than `4°` from CAD
- retry with a narrower locality window
- keep the better result only

This is intentionally narrow. It is not a family-wide search-radius heuristic.
